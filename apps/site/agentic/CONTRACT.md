# Agentic layer — build contract

How the RamosLabs DS agentic layer is produced, and the rules for the distilled content
it inlines. Read this before editing doctrine or the generator.

## Goal

Make the design system consumable by AI coding agents via `WebFetch`. The build emits,
into the Astro output (`apps/site/dist/`) and refreshes committed copies at the repo root:

- `llms.txt` — concise index (llmstxt.org standard)
- `llms-full.txt` — single self-contained document, every page inlined + the full token table
- `registry.json` — machine-readable registry (tokens + patterns; `components` empty until the Vue library ships)
- `robots.txt` — allows all crawlers plus explicit welcome blocks for AI agents, links the sitemap
- `tokens.json` — the flat DTCG token map (copied from `@ramoslabs/tokens`)
- `AGENTS.md` — hand-authored onboarding, copied from the repo root

`sitemap-index.xml` is generated by `@astrojs/sitemap` (route-aware, respects `site` +
`trailingSlash`). Per-page SEO/social head metadata (description, Open Graph, Twitter card,
canonical, JSON-LD) is emitted by `apps/site/src/components/BaseHead.astro`; Astro owns the
document `<title>`.

## How it is generated

`llms.txt`, `llms-full.txt`, `registry.json`, `robots.txt` are produced by the Astro
integration `apps/site/agentic/integration.ts`, which runs on the `astro:build:done` hook.
It reads:

- the built `@ramoslabs/tokens` JSON (`packages/tokens/dist/tokens.json`) for the token table,
- the page list + metadata from `apps/site/src/data/pages.ts`,
- the distilled bodies from `apps/site/agentic/content/**` (see below).

The production URL comes from Astro's `site` config (`https://design.ramoslabs.com`) — the
single source of truth. Every page link in every artifact is a clean, crawlable URL
(`/foundations/colors/`), never a query-param route.

## Page metadata: `src/data/pages.ts`

The ordered page list. Each entry:

```ts
type Page = {
  id: string          // kebab-case, unique
  title: string
  group: 'root' | 'foundations' | 'patterns'
  path: string        // clean URL with leading + trailing slash, e.g. '/foundations/colors/'
  summary: string     // one line an agent reads to decide relevance
  icon: string        // Material Symbols ligature (sidebar)
  contentFile: string // relative to apps/site/agentic/content, e.g. 'foundations/Colors.md'
  interactive?: boolean // true if the page ships a JS island (excluded from agentic prose)
}
```

`SITE.summary`/`SITE.tagline` and the copyright/attribution constants also live here, so the
site, the SEO head, and the agentic layer all read one source.

## Distilled content file contract

One markdown file per doc page under `apps/site/agentic/content/<group>/<PageId>.md`
(Introduction at `apps/site/agentic/content/Introduction.md`). Each file is YAML frontmatter
then the distilled body; the integration inlines the body into `llms-full.txt`.

```
---
id: colors
title: Colors
group: foundations
summary: One sentence an agent reads to decide if this page is relevant.
---

<distilled markdown body>
```

The authoritative `summary` for output is the one in `src/data/pages.ts`; keep the
frontmatter `summary` in sync as a convenience copy.

### Distillation rules (correctness is non-negotiable)

- DISTILL, never invent. Compress and strip demos, but never add, drop, or alter a rule,
  token name, hex value, ratio, px value, or threshold. Every prescriptive statement must be
  traceable to the source `.astro` page.
- Strip presentation: layout markup, inline styles, icons, live demos. Keep prose, rules,
  tables, and code recipes.
- Preserve the DS framing verbatim in meaning: keep `Rule:` / `Why:` lines and the
  `✓ Recommended` / `✕ Avoid` structure. Never rewrite these as "Do / Don't".
- Keep token references by their token NAME (e.g. `--color-primary`, `--shadow-focus`), not
  the resolved value, except where the source states a raw hex/px as canonical (e.g. mono-indigo `#4f46e5`).
- Convert a live demo into a short prose description + a copyable code recipe when it teaches
  something an agent needs (e.g. the `accent-color` form recipe). Drop decorative demos.
- Output must pass the DS voice rules: no em-dash or en-dash (use a comma, colon, or period);
  English; state the current system as truth, no before/after or "previously" self-history.
- Target ~40-60% of the source prose length. Lean, hook-led, agent-actionable.

## Keeping doctrine and distilled files in sync

When you change doctrine on a page (`apps/site/src/pages/**/*.astro`), update its distilled
file in `apps/site/agentic/content/**` in the same change (same rule, token name, hex, ratio,
px, or threshold, never invented or dropped). The next `bun run --filter site build`
regenerates the artifacts via the integration. A doctrine change that skips the distilled
update ships an agentic layer that contradicts the site.
