    <span class="property">pointer-events</span><span class="punctuation">:</span> <span class="value">none</span><span class="punctuation">;</span>
<span class="punctuation">}</span>
<span class="selector">.control:active::after</span> <span class="punctuation">{</span> <span class="property">opacity</span><span class="punctuation">:</span> <span class="value">var(--state-pressed)</span><span class="punctuation">;</span> <span class="punctuation">}</span> <span class="comment">/* every input */</span>`}} />
  </div>
</div>

{/* ============================================ */}
{/* 3 - THE HOVER GATE                            */}
{/* ============================================ */}

<div className="ix-section">
  <div className="ix-section-header">
    <span className="ix-section-number">3</span>
    <h2 className="ix-section-title">Hover belongs to pointers</h2>
  </div>

  <p className="ix-text">
    Wrap every hover rule in <code>@media (hover: hover) and (pointer: fine)</code>. It fires only where a pointer can actually rest over a control.
  </p>

  <div className="ix-callout">
    <span className="ix-material" aria-hidden="true">warning</span>
    <p className="ix-callout-text">
      <strong>The sticky hover bug.</strong> A touchscreen has no pointer to move away, so it never clears an ungated hover. Tap one and the style <strong>sticks</strong>: buttons stay lit, cards freeze mid transition. The gate is the fix, not a nicety.
    </p>
  </div>

  <div className="ix-code">
    <div className="ix-code-header">
      <span className="ix-code-dot red" aria-hidden="true"></span>
      <span className="ix-code-dot yellow" aria-hidden="true"></span>
      <span className="ix-code-dot green" aria-hidden="true"></span>
      <span className="ix-code-lang">CSS</span>
    </div>
    <pre dangerouslySetInnerHTML={{__html: `<span class="comment">/* Press works everywhere, so it lives at the top level. */</span>
<span class="selector">.control:active::after</span> <span class="punctuation">{</span> <span class="property">opacity</span><span class="punctuation">:</span> <span class="value">var(--state-pressed)</span><span class="punctuation">;</span> <span class="punctuation">}</span>

<span class="comment">/* Hover is a pointer-only enhancement. */</span>
<span class="selector">@media (hover: hover) and (pointer: fine)</span> <span class="punctuation">{</span>
    <span class="selector">.control:hover::after</span> <span class="punctuation">{</span> <span class="property">opacity</span><span class="punctuation">:</span> <span class="value">var(--state-hover)</span><span class="punctuation">;</span> <span class="punctuation">}</span>
<span class="punctuation">}</span>`}} />
  </div>
</div>

{/* ============================================ */}
{/* 4 - THE FOCUS RING                            */}
{/* ============================================ */}

<div className="ix-section">
  <div className="ix-section-header">
    <span className="ix-section-number">4</span>
    <h2 className="ix-section-title">A ring you can always see</h2>

    Style <code>:focus-visible</code>, not <code>:focus</code>: the ring shows for the keyboard and hides after a plain click. Color it with <code>--shadow-focus</code>, an indigo ring that reads on any surface. A white ring vanishes the moment the background is not white.

{/* ============================================ */}
{/* 5 - THE RULES                                 */}
{/* ============================================ */}

<div className="ix-section">
  <div className="ix-section-header">
    <span className="ix-section-number">5</span>
    <h2 className="ix-section-title">The rules</h2>
  </div>

  <div className="ix-dodont">
    <div className="ix-list do">
      <div className="ix-list-head">
        <span className="ix-material" style={{ fontSize: '1.25rem' }} aria-hidden="true">check_circle</span>
        Do
      </div>
      <ul>
        <li>Design <code>:active</code> first, then <code>:focus-visible</code>, then hover.</li>
        <li>Wrap every hover in <code>@media (hover: hover) and (pointer: fine)</code>.</li>
        <li>Express state with tonal indigo via <code>--state-*</code> opacity.</li>
        <li>Ring with <code>--shadow-focus</code> on <code>:focus-visible</code> only.</li>
        <li>Gate transforms behind <code>prefers-reduced-motion</code>, and let color carry the meaning without them.</li>
        <li>Give controls a 6px radius, <code>--radius-sm</code>, and pair each state with its ARIA.</li>
      </ul>
    </div>
    <div className="ix-list dont">
      <div className="ix-list-head">
        <span className="ix-material" style={{ fontSize: '1.25rem' }} aria-hidden="true">cancel</span>
        Don't
      </div>
      <ul>
        <li>Build feedback on hover alone. Touch gets nothing.</li>
        <li>Leave hover ungated. It sticks after a tap.</li>
        <li>Lift a card or swap its shadow to signal hover. Shadow is elevation, not state.</li>
        <li>Style <code>:focus</code> when you mean <code>:focus-visible</code>.</li>
        <li>Ring in <code>white</code>. It vanishes off a white surface.</li>
        <li>Put a pill radius on a wide button.</li>
      </ul>
    </div>
  </div>
</div>
