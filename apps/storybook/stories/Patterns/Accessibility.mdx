  </div>

  <p className="a11y-section-text">
    Screen readers interpret your HTML. Use <strong>semantic elements first</strong>: <code>&lt;button&gt;</code> not <code>&lt;div onclick&gt;</code>, <code>&lt;nav&gt;</code> not <code>&lt;div class="nav"&gt;</code>. Native HTML comes with built-in accessibility, roles, keyboard handling, focus management. Don't reinvent what the browser gives you for free.
  </p>

  <p className="a11y-section-text" style={{ marginTop: '1rem' }}>
    <strong>ARIA Live Regions</strong> announce dynamic changes to screen reader users. When the cart count updates, when a form shows an error, when search results load, these moments need to be announced. Use <code>role="alert"</code> for urgent errors (interrupts immediately) and <code>role="status"</code> for polite updates (waits for a pause).
  </p>

  <p className="a11y-section-text" style={{ marginTop: '1rem' }}>
    The golden rule: <strong>No ARIA is better than bad ARIA</strong>. If you're unsure, stick with semantic HTML. A <code>&lt;button&gt;</code> is always better than <code>&lt;div role="button" tabindex="0"&gt;</code>.
  </p>
