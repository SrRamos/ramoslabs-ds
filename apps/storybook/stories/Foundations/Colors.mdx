When only one color can say "act here," users learn it in seconds and every screen stays calm. This is restraint by design, not a lack of range: the depth comes from a wide neutral ramp and semantic feedback colors, not from competing brand hues. The accent is <strong>Indigo 600 (#4f46e5)</strong> and there is no rainbow, no second voice fighting for attention.

    <div className="col-callout-bar"></div>
    <p className="col-callout-body">
      <strong>On the horizon: APCA.</strong> The WCAG 2.x ratio treats contrast as a simple luminance division, which is known to misjudge readability at the extremes (it is over-strict on dark text and over-lenient on light text). The <strong>Accessible Perceptual Contrast Algorithm (APCA)</strong>, drafted for WCAG 3.0, models perceived lightness, text weight, and polarity instead, producing an Lc value from roughly -108 to 108 [4]. WCAG 3.0 is still an early W3C Working Draft and APCA is not yet a normative requirement [4b], so <strong>WCAG 2.2 remains the floor we certify against</strong>. We track APCA now so the palette is ready when it lands.
    </p>

  <h2 className="col-section-title">4. Dark mode taxonomy</h2>
  <p className="col-section-intro">
    The current token set is light-first. Dark mode is not shipped as values yet, so the rules below are the <strong>taxonomy for how it will extend</strong>, drawn from primary guidance, so a future dark theme stays inside the system. Roles do not change; only what each role points at changes, behind a <code>[data-theme="dark"]</code> selector.
  </p>

  <div className="col-dark" style={{ background: '#0f172a', padding: '1rem', borderRadius: '14px' }}>

      <div className="col-dark-hex">background ≈ #0f172a / #121212</div>

      <div className="col-dark-name">Elevation = lighter surface</div>
      <div className="col-dark-hex">surface ≈ slate 800</div>
      <div className="col-dark-note">In the dark, higher surfaces get lighter, not darker. Elevation is expressed by tone, not just shadow [6][7].</div>

    <div className="col-dark-cell" style={{ background: '#334155' }}>

      <div className="col-dark-hex">primary ≈ indigo 400 (#818cf8)</div>
      <div className="col-dark-note">Saturated indigo 600 vibrates on dark. Step up the ramp to a lighter, calmer accent [6].</div>

      <div className="col-dark-note">Flip the neutral ramp: light text on dark, and keep the same 4.5:1 / 7:1 floors.</div>

      <div className="col-dark-name">Honor the OS</div>
      <div className="col-dark-hex">@media (prefers-color-scheme: dark)</div>

<div className="col-section">
  <h2 className="col-section-title">5. Applying color: the rules</h2>
  <p className="col-section-intro">
    A decision framework for a new project. Follow the 60-30-10 balance, indigo as the 10% accent, neutrals as the 60% + 30%, so the accent keeps its meaning [9]. Then obey these rules.
  </p>

  <div className="col-rules">
    <div className="col-rule col-rule--do">
      <div className="col-rule-head">Do</div>
      <ul>
        <li>Consume <strong>semantic roles</strong> in product code: <code>var(--color-primary)</code>, <code>var(--color-error-surface)</code>. Roles are the contract.</li>
        <li>Derive interaction states by walking the ramp: rest = <code>--color-primary</code> (600), hover / active = <code>--color-primary-dark</code> (700), tinted zone = <code>--color-primary-surface</code> (50).</li>
        <li>Use the full <strong>feedback triad</strong> (solid + surface + text) for every status message, and pair it with an icon and a label.</li>
        <li>Keep normal text at <strong>4.5:1+</strong> and UI boundaries / meaningful icons at <strong>3:1+</strong>. Verify with the Color Picker story before shipping.</li>
        <li>Reach for <code>--color-error-strong</code> when white text sits on red; reach for <strong>dark</strong> text on amber.</li>
        <li>Name any new token <code>--color-&lt;role&gt;[-&lt;variant&gt;]</code>, where variant is one of <code>surface</code> / <code>border</code> / <code>text</code> / <code>strong</code> / <code>light</code> / <code>dark</code> / <code>hover</code>.</li>
      </ul>
    </div>
    <div className="col-rule col-rule--dont">
      <div className="col-rule-head">Don't</div>
      <ul>
        <li>Hardcode a hex or reference a <strong>primitive</strong> (<code>--color-indigo-600</code>) in a component. Primitives feed roles, not UI.</li>
        <li>Introduce a second accent hue to "add energy". If it needs to stand out, it is indigo. Violet is decorative support only.</li>
        <li>Signal state with color alone, or remove the focus ring without a stronger replacement (violates 1.4.1 / 1.4.11).</li>
        <li>Put white text on <code>--color-warning</code> (2.15:1) or <code>--color-error</code> red 500 for body copy (3.76:1).</li>
        <li>Repurpose <code>--color-info</code> as a call to action. Info is context; indigo is action. Keeping them distinct is the point.</li>
        <li>Use <code>--color-text-disabled</code> for anything a user must read. It sits below the floor on purpose.</li>
      </ul>
    </div>
  </div>
</div>
