      <div className="rs-compare-name">Container query</div>
      <div className="rs-compare-q">@container (min-width: 24rem)</div>
      <ul className="rs-compare-list">
        <li>Asks how much space this element was actually given.</li>
        <li>Right for reusable parts: cards, tiles, widgets that appear in many slots.</li>
        <li>The parent opts in with <code>container-type</code>; children size to <code>cqw</code> units.</li>
        <li>The component becomes portable: drop it anywhere and it adapts.</li>
      </ul>
    </div>
  </div>

  <div className="rs-callout">
    <div className="rs-callout-bar"></div>
    <p className="rs-callout-body">
      <span className="rs-callout-key">Rule</span> Match the query to its scope: a media query against a token for the page shell, a container query for any part that must work in more than one slot.
      <span className="rs-callout-why"><span className="rs-callout-key">Why</span> A media query asks how wide the viewport is, which is what the shell responds to. A container query asks how much room an element was actually given, which is what a portable component responds to.</span>
    </p>
  </div>

  <div className="rs-ry">
    <div className="rs-ry-item rec">
      <span className="rs-ry-icon" aria-hidden="true">&#10003;</span>
      <div>
        <span className="rs-ry-label">Recommended</span>
        <span className="rs-ry-text">For a capability or preference, dark mode, reduced motion, a coarse pointer, reach for a media query.</span>
      </div>
    </div>
    <div className="rs-ry-item avoid">
      <span className="rs-ry-icon" aria-hidden="true">&#10005;</span>
      <div>
        <span className="rs-ry-label">Avoid</span>
        <span className="rs-ry-text">Reaching for a container query there. A container knows only its own size, never color scheme, motion preference, or pointer type.</span>
      </div>
    </div>
  </div>
</div>

<div className="rs-section">
  <span className="rs-num">04</span>
  <h2 className="rs-h2">The best breakpoint is the one you never wrote</h2>
  <p className="rs-intro">
    Between the structural thresholds, layout should flow, not jump. Intrinsic layouts lean on what CSS already does: a grid that fits as many columns as comfortably fit, type and spacing that scale as a range. The result adapts at <strong>every</strong> width, not just five, with far fewer media queries.
  </p>

  <div className="rs-demo">
    <div className="rs-fluid-grid">
      <div className="rs-fluid-card"><span className="rs-fluid-card-title">reflow</span><span className="rs-fluid-card-text">Cards wrap on their own as space allows.</span></div>
      <div className="rs-fluid-card"><span className="rs-fluid-card-title">auto-fit</span><span className="rs-fluid-card-text">Column count follows the container width.</span></div>
      <div className="rs-fluid-card"><span className="rs-fluid-card-title">minmax</span><span className="rs-fluid-card-text">Each column stays between a floor and a fair share.</span></div>
      <div className="rs-fluid-card"><span className="rs-fluid-card-title">zero queries</span><span className="rs-fluid-card-text">No breakpoint wrote this. The grid did.</span></div>
    </div>
    <p className="rs-demo-caption">
      Live, one rule, no media query: <code>grid-template-columns: repeat(auto-fit, minmax(160px, 1fr))</code>. Narrow the docs panel and the columns reflow on their own. Reach for this before any breakpoint.
    </p>
  </div>

  <div className="rs-demo" style={{ marginTop: '1.25rem' }}>
    <p className="rs-fluid-title">Fluid type scales with the screen</p>
    <p className="rs-demo-caption">
      This heading is <code>clamp(1.5rem, 3vw + 1rem, 2.75rem)</code>: it grows and shrinks smoothly, held between a floor and a ceiling. Use <code>clamp()</code>, <code>min()</code>, and <code>max()</code> for type, spacing, and measure to erase the step changes a stack of breakpoints would cause.
    </p>
  </div>

  <div className="rs-callout">
    <div className="rs-callout-bar"></div>
    <p className="rs-callout-body">
      <span className="rs-callout-key">Rule</span> Reach for fluid sizing and an intrinsic grid before you write any breakpoint.
      <span className="rs-callout-why"><span className="rs-callout-key">Why</span> An auto-fit grid and clamped type adapt at every width, not just at the five thresholds, and with far fewer queries to keep in sync.</span>
    </p>
  </div>

  <div className="rs-ry">
    <div className="rs-ry-item rec">
      <span className="rs-ry-icon" aria-hidden="true">&#10003;</span>
      <div>
        <span className="rs-ry-label">Recommended</span>
        <span className="rs-ry-text">Let the layout flow: <code>repeat(auto-fit, minmax(160px, 1fr))</code> for columns, <code>clamp()</code> for type and spacing.</span>
      </div>
    </div>
    <div className="rs-ry-item avoid">
      <span className="rs-ry-icon" aria-hidden="true">&#10005;</span>
      <div>
        <span className="rs-ry-label">Avoid</span>
        <span className="rs-ry-text">Stacking breakpoints to imitate that flow. It steps between thresholds and leaves more queries to maintain.</span>
      </div>
    </div>
  </div>
</div>

<div className="rs-section">
  <span className="rs-num">05</span>
  <h2 className="rs-h2">Safe areas and notches</h2>
  <p className="rs-intro">
    On modern phones the screen is not a clean rectangle. Notches, rounded corners, and the home indicator carve in, and a layout drawn to the physical edge can bury a primary action under a system gesture. Honor the insets for anything anchored to a screen edge, and <strong>add</strong> the inset to your spacing token rather than replacing it.
  </p>

  <div className="rs-safearea">
    <div className="rs-phone" aria-hidden="true">
      <div className="rs-phone-notch"></div>
      <div className="rs-phone-safe">safe area</div>
      <div className="rs-phone-cta">Get tickets</div>
      <div className="rs-phone-indicator"></div>
    </div>
    <div>
      <p className="rs-intro" style={{ margin: 0 }}>
        The dashed inset is the safe area. The button clears the home indicator because its padding is <code>calc(var(--space-4) + env(safe-area-inset-bottom))</code>, not a bare token. Insets are unlocked by <code>viewport-fit=cover</code>; on a device with none, <code>env()</code> resolves to zero and spacing is unchanged.
      </p>
      <p className="rs-safearea-note">
        Thumb-reach and target sizing for these anchored controls live in Foundations / Spacing. This page claims only the responsibility: keep every screen-anchored control clear of system chrome.
      </p>
    </div>
  </div>

  <div className="rs-callout">
    <div className="rs-callout-bar"></div>
    <p className="rs-callout-body">
      <span className="rs-callout-key">Rule</span> Honor the safe-area insets for anything anchored to a screen edge.
      <span className="rs-callout-why"><span className="rs-callout-key">Why</span> Notches, rounded corners, and the home indicator carve into the screen, so a control drawn to the physical edge can land under a system gesture.</span>
    </p>
  </div>

  <div className="rs-ry">
    <div className="rs-ry-item rec">
      <span className="rs-ry-icon" aria-hidden="true">&#10003;</span>
      <div>
        <span className="rs-ry-label">Recommended</span>
        <span className="rs-ry-text">Add the inset to your token: <code>calc(var(--space-4) + env(safe-area-inset-bottom))</code>. Where a device has no insets, <code>env()</code> resolves to zero and the spacing is unchanged.</span>
      </div>
    </div>
    <div className="rs-ry-item avoid">
      <span className="rs-ry-icon" aria-hidden="true">&#10005;</span>
      <div>
        <span className="rs-ry-label">Avoid</span>
        <span className="rs-ry-text">Replacing the token with <code>env()</code> alone, or drawing to the physical edge. Where there are no insets the spacing collapses to zero and the control can fall under system chrome.</span>
      </div>
    </div>
  </div>
