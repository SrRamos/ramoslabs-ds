# Agentic layer — build contract

Shared contract for every agent building the RamosLabs DS agentic layer. Read this
fully before writing anything. The pieces must interlock exactly.

## Goal

Make the design system consumable by AI coding agents via `WebFetch`. Four artifacts,
served from the deployed Cloudflare Pages site root and committed at repo root:

- `llms.txt` — concise index (llmstxt.org standard)
- `llms-full.txt` — single self-contained document, everything inlined
- `AGENTS.md` — onboarding for an agent that BUILDS UI consuming this DS
- `registry.json` — machine-readable registry (tokens + patterns; components slot empty until F1)

`llms.txt`, `llms-full.txt`, `registry.json` are GENERATED by
`apps/storybook/agentic/build-agentic.mjs` from: the built `@ramoslabs/tokens` JSON +
the distilled content files below + `manifest.ts`. `AGENTS.md` is hand-authored.

## Distilled content file contract

One markdown file per book page, authored by hand from the source MDX.

- Path: `apps/storybook/agentic/content/<group>/<PageId>.md`
  - `<group>` is `foundations` or `patterns`
  - `<PageId>` is the source MDX basename (e.g. `Colors`, `FormElements`, `TokenReference`)
  - Introduction goes at `apps/storybook/agentic/content/Introduction.md` (no group)
- Each file MUST start with YAML frontmatter, then the distilled body:

```
---
id: colors
title: Colors
group: foundations
storybookTitle: Foundations/Colors
summary: One sentence an agent reads to decide if this page is relevant.
---

<distilled markdown body>
```

- `id`: kebab-case, unique, derived from PageId.
- `storybookTitle`: the EXACT string from the page's `<Meta title="..." />` in the MDX.
  The generator turns this into the live docs URL, so copy it verbatim.
- `summary`: one line, no trailing period required, plain text.

### Distillation rules (correctness is non-negotiable)

- DISTILL, never invent. Compress and strip JSX/demos, but never add, drop, or alter a
  rule, token name, hex value, ratio, px value, or threshold. Every prescriptive
  statement must be traceable to the source MDX.
- Strip all JSX: `import` lines, `<Meta>`, live demo components, React/Vue expressions.
  Keep only prose, rules, tables, and code recipes.
- Preserve the DS framing verbatim in meaning: keep `Rule:` / `Why:` lines and the
  `✓ Recommended` / `✕ Avoid` structure. Never rewrite these as "Do / Don't".
- Keep token references by their token NAME (e.g. `--color-primary`, `--shadow-focus`),
  not by resolved value, except where the source explicitly states a raw hex/px as the
  canonical value (e.g. mono-indigo `#4f46e5`).
- Convert live demos into a short prose description + a copyable code recipe when the
  demo teaches something an agent needs (e.g. the `accent-color` form recipe). Drop
  purely decorative demos.
- Output must itself pass the DS voice rules: no em-dash or en-dash (use a comma, colon,
  or period); English; state the current system as truth, no before/after or "previously"
  self-history.
- Target ~40-60% of the source prose length. Lean, hook-led, agent-actionable.

## manifest.ts contract

`apps/storybook/agentic/manifest.ts` exports the ordered page list the generator walks.
Shape (TypeScript, `type` not `interface`):

```ts
export type Page = {
  id: string
  title: string
  group: 'foundations' | 'patterns' | 'root'
  storybookTitle: string
  file: string // path relative to agentic/content, e.g. 'foundations/Colors.md'
}
export const SITE_URL = process.env.SITE_URL ?? 'https://ramoslabs-ds.pages.dev'
export const pages: Page[] = [ /* Introduction first, then foundations in book order, then patterns */ ]
```

The generator derives each page's live docs URL from `storybookTitle`:
`Foundations/Colors` -> `${SITE_URL}/?path=/docs/foundations-colors--docs`
(lowercase, `/` and spaces -> `-`, strip other punctuation).

## build-agentic.mjs contract

Node ESM script, run from `apps/storybook`. Steps:

1. Import token data from `@ramoslabs/tokens/json` (the flat DTCG JSON) to build the
   full token reference table (grouped by category, name + value).
2. Read each `manifest.pages` content file, parse frontmatter (a tiny hand-rolled parser
   is fine, do NOT add a dependency), collect summaries + bodies.
3. Emit to REPO ROOT and copy into `apps/storybook/storybook-static/`:
   - `llms.txt`: H1 title, blockquote summary (from Introduction), then a `## Foundations`
     and `## Patterns` section, each a bullet list of `[Title](docsURL): summary`, plus a
     `## Machine-readable` section linking `llms-full.txt`, `registry.json`, and the token JSON.
   - `llms-full.txt`: title + intro, the full token reference table, then every page body
     inlined under `# <Title>` headings, in manifest order.
   - `registry.json`: `{ name, version, generatedFrom, tokens: { total, byCategory, url },
     patterns: [{ id, title, url, summary }], components: [] }`. Valid JSON, no comments.
4. Never write into git-ignored-only locations; root artifacts are committed.

Wire an `agentic:build` script in `apps/storybook/package.json`. The Cloudflare Pages
build command runs Storybook build first, then `agentic:build`, so artifacts land in the
fresh `storybook-static/`.
